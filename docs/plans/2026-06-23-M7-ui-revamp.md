# M7 GOAL тАФ UI/UX revamp: "The Reading Room" agent workspace (sidebar ┬╖ theme ┬╖ pipeline-hero ┬╖ evidence-in-context)

Status: APPROVED тАФ aesthetic = A "Reading Room"; scope = full aтАУe, autonomous ┬╖ Date: 2026-06-23 ┬╖ Depends on: M0тАУM6╬╡ (`main`, 126 commits / 178 tests)

## Why
The app works but looks generic and hides its best capabilities. Concretely, `index.html`'s global `<style>` uses
**Inter**, **light-only** (`color-scheme: light`), and a **blue/teal gradient on near-white** with hardcoded hex тАФ
the exact "AI-slop" aesthetic to avoid. Meanwhile `plan_and_check` (the M6╬│/╬╡ evidence map + thesis verdict тАФ the
literature-review hero) **isn't wired to the UI at all**, counter-evidence is computed but hidden, and which judge
produced a verdict is invisible (and the default mock judge misleads). Reference apps (KunAgent/Kun, op7418/CodePilot)
point the direction: **a sidebar agent-workspace that makes the agent's pipeline the UX hero**.

## Aesthetic direction тАФ commit (refined scholarly instrument, NOT flashy)
**"The Reading Room"** тАФ a precise, editorial research instrument. Restraint + craft, not maximalism.
- **Typography** (bundled offline, no runtime CDN): **Fraunces** (optical-serif display/headings тАФ characterful,
  scholarly), **Hanken Grotesk** (warm grotesque body/UI sans), **IBM Plex Mono** (locators, trace, data). Kills Inter.
- **Color** тАФ one dominant + sharp semantic accents, both themes via CSS variables:
  - Light: warm paper bg (`#FBFAF7`-ish), ink text (`#1A1A17`), hairline rules, a single **ink-indigo** primary accent.
  - Dark: deep warm charcoal (`#16150F`-ish), parchment text тАФ same accents retuned.
  - Semantic (desaturated, refined тАФ never neon): support = forest green, contradict = oxblood, contested = ochre/amber,
    unclear/unrelated = slate. Kills the blue/teal gradient.
- **Motion** (restrained, high-impact only): staggered reveal of evidence cards on result (`animation-delay`), a soft
  pulse on the active pipeline stage, a smooth cross-fade on theme switch. No scattered micro-jitter.
- **Signature element**: the **agent pipeline** (plan тЖТ retrieve тЖТ judge тЖТ report) as an elegant process strip with the
  live ┬з10 trace under it тАФ the one memorable thing, and the most on-brand "Agent Harness" statement.
- **Texture/depth**: flat + precise тАФ hairline editorial rules, minimal shadows, optional faint paper grain.

## Engineering reality (what this touches)
- **Renderer-only + ONE small backend gap**: all visual work is in `electron/`. The single non-renderer change is
  wiring **`plan_and_check` through the IPC** (`electron/preload.ts` + `electron/main.ts` + `electron/renderer/api.d.ts`
  тАФ the worker already handles it; the bridge is missing). `src/**` core stays untouched (**iron rule holds**).
- **Theme system**: replace the `index.html` `<style>` globals + the inline-styled tabs (App/EvalTrace/Settings, and
  Library/Sources/Matrix as needed) with a **tokens stylesheet** (CSS variables, light + dark) + a small `ThemeProvider`
  (persisted via `localStorage`). `build.mjs` must bundle the CSS + `@font-face` `.woff2` files (offline).
- **Offline**: fonts are bundled `.woff2` (a packaged desktop app has no CDN). No `position:fixed` traps; respects
  `prefers-reduced-motion`.
